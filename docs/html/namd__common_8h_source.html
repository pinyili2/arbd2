143// Setting this define to a non-zero value will cause the
144// HomePatches to separate the hydrogen groups in their
145// HomePatch::atom lists (all water molecules first, in arbitrary
146// order, followed by all non-waters, in arbitrary order).

149// DMK - Atom Sort
150// Setting this define to a non-zero value will cause the nonbonded compute
151// objects (pairs, not selfs) to sort the atoms along a line connecting the
152// center of masses of the two patches. This is only done during timesteps
153// where the pairlists are being generated. As the pairlist is being
154// generated, once an atom that is far enough away along the line is found,
155// the remaining atoms are automatically skipped (avoiding a distance
156// calculation/check for them).
157// NOTE: The "less branches" flag toggles between two versions of merge sort.
158// When it is non-zero, a version that has fewer branches (but more integer
159// math) is used. This version may or may not be faster or some architectures.
160#define NAMD_ComputeNonbonded_SortAtoms 0
161 #define NAMD_ComputeNonbonded_SortAtoms_LessBranches 1
